# Contributing

Thanks for your interest in contributing.

## Workflow

1. Open an issue describing the change before large work.
2. Fork and create a branch named `type/short-description` (for example `fix/handle-null-input`).
3. Keep changes focused; one logical change per pull request.
4. Ensure the build, linters, and tests pass locally before opening a PR.
5. Use clear, conventional commit messages (`feat:`, `fix:`, `docs:`, `chore:`, `test:`).

## Pull Requests

- Describe what changed and why.
- Link the related issue.
- Add or update tests and documentation for behavior changes.

## Code of Conduct

By participating you agree to abide by the [Code of Conduct](CODE_OF_CONDUCT.md).
